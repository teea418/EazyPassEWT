# EazyPassEWT

> 升学e网通 (ewt360.com) 自动化学习辅助工具
>
> 基于 Playwright 模拟浏览器操作，自动登录平台、进入学习计划、逐条播放未完成的视频任务，播放期间自动处理弹窗，全程静默或可视运行。

## 功能特性

- **自动登录** —— 读取环境变量凭据，自动完成密码输入与协议勾选
- **智能选择** —— 自动识别当天未完成的视频任务，支持多人学习计划选择
- **倍速播放** —— 随机 1.8~2.5 倍速（每次不同），降低模式识别风险
- **反检测保护** —— 隐藏 `navigator.webdriver` 标志，轮询间隔随机抖动
- **弹窗处理** —— 播放期间自动检测并点击确认弹窗，无需人工干预
- **超时保护** —— 视频播放设硬上限超时，防止死循环
- **断网重试** —— 网络波动时指数退避自动重试（3 次）
- **日志记录** —— 每次运行生成独立日志文件，支持用户名脱敏输出
- **静默模式** —— 支持无头模式后台运行，不干扰桌面操作

## 快速开始

### 前置条件

- Python 3.10+
- Windows / macOS / Linux

### 安装

```bash
# 克隆仓库
git clone https://github.com/teea418/FuckEWT.git
cd FuckEWT

# 创建虚拟环境（推荐）
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS / Linux:
# source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 安装 Playwright 浏览器
playwright install chromium
```

### 配置

```bash
# 复制配置文件模板
cp .env.example .env
```

编辑 `.env` 文件，填入真实凭据：

```env
# 升学e网通 登录账号
EWT_USERNAME=your_username_here

# 升学e网通 登录密码
EWT_PASSWORD=your_password_here

# 登录页面地址（一般无需修改）
EWT_URL=https://www.ewt360.com/

# 作业列表页面地址（一般无需修改）
EWT_HOMEWORK_URL=https://teacher.ewt360.com/ewtbend/bend/index/index.html#/student/homework
```

> **安全提示**：`.env` 已加入 `.gitignore`，不会提交到仓库。请勿将真实凭据分享或提交至任何公开渠道。

### 运行

```bash
python main.py
```

启动后按提示操作：
1. 阅读并确认风险提示
2. 选择是否显示浏览器界面（`y` 显示 / `n` 后台静默）
3. 如有多个学习计划，选择要执行的一项
4. 工具将自动检测待完成视频并开始播放

## 技术原理

| 模块 | 技术方案 | 说明 |
|------|---------|------|
| 浏览器自动化 | Playwright (Chromium) | 模拟真实用户操作，支持无头模式 |
| 反检测 | `--disable-blink-features=AutomationControlled` | 隐藏自动化标志，降低被识别为机器人的概率 |
| 登录流程 | CSS 选择器定位 + `.fill()` | 读取环境变量凭据，自动勾选协议后提交 |
| 任务识别 | DOM 属性解析 | 通过 `data-active`、`data-type`、`data-finish` 属性定位任务与视频 |
| 视频播放 | JavaScript 注入 | `video.playbackRate` 随机 1.8~2.5x 倍速，`video.muted` 静音，轮询 `currentTime` 检测进度 |
| 行为拟人化 | 随机抖动 + 随机间隔 | 轮询间隔 200~600ms 抖动，视频间 2~8s 随机延迟 |
| 容错机制 | 指数退避重试 | `retry_call()` 通用重试工具，延迟 = `base * backoff^(n-1)` |

## 安全与风险降低措施

### 技术安全

- **凭据管理**：账号密码存储在本地 `.env` 文件，已加入 `.gitignore`，仅用于 Playwright 的浏览器输入操作
- **日志脱敏**：密码输出为 `******`，用户名仅保留首尾各 2 位（如 `te****18`）
- **空值校验**：账号或密码为空时直接退出，避免无效登录
- **无数据外传**：工具仅与 ewt360.com 平台交互，不向任何第三方发送数据
- **安全退出**：`quit_ewt()` 函数对未初始化的浏览器对象做判空处理，防止异常崩溃
- **限流保护**：内置 3 次指数退避重试 + 视频间 2~8s 随机延迟，避免请求风暴

### 反检测

- 启动 Chromium 时添加 `--disable-blink-features=AutomationControlled` 参数，隐藏 `navigator.webdriver` 标志
- 播放轮询间隔加入 200~600ms 随机抖动
- 视频间插入 2~8s 随机延迟，模拟人工切换
- 倍速在 1.8~2.5x 区间随机，避免固定模式

## 法律风险分析与防御策略

> 以下内容为基于公开法律信息的技术分析，不构成法律意见。如有法律疑问，请咨询专业律师。

### 平台用户协议中的潜在无效条款

本工具存档了升学e网通当前的用户协议（见 `用户协议.txt`）。经分析，该协议存在 **多处违反《中华人民共和国民法典》格式条款规定** 的情形：

| 条款 | 内容 | 法律问题 |
|------|------|---------|
| **第1条/第49条** 单方修改协议 | 平台可随时修改协议，公布即生效，继续使用即视为同意 | 违反《民法典》第496条——格式条款提供方未尽合理提示说明义务 |
| **第37条** 账号泄露完全免责 | "升学e网通不承担任何责任" | 违反《民法典》第497条——不合理免除自身责任；违反《电子商务法》第27条（安全保障义务） |
| **第47条** 过度索取用户作品权利 | 用户授予平台使用、复制、修订、改写、发布、翻译、分发、展示用户的作品 | 违反《民法典》第497条——不合理扩大自身权利、限制对方权利 |
| **第61-66条** AI服务完全免责 | 对AI生成内容导致的所有损失不承担任何责任 | 与《生成式人工智能服务管理暂行办法》第12条精神相悖 |
| **第75条** AI输出知识产权归属 | 用户使用AI生成的内容知识产权归平台，且用户授予永久免费全球可转让许可 | 违反公平原则，可能构成显失公平（《民法典》第151条） |
| **第91条** 指定管辖法院 | "向杭州市拱墅区人民法院提起诉讼" | 对异地用户增加不合理维权成本，消费者可主张该条款无效（《最高人民法院关于适用〈中华人民共和国民事诉讼法〉的解释》第31条） |

### 核心法律观点

**《民法典》第497条** 明确规定：提供格式条款一方不合理地免除或者减轻其责任、加重对方责任、限制对方主要权利的，该格式条款无效。

这意味着：

1. **平台不能仅凭用户协议中的单方条款就对用户随意封号而不承担责任**——尤其是当封号导致用户已购买的付费服务无法使用时
2. **如果平台以"违反协议"为由封禁账号**，用户有权要求平台 **证明该封禁行为具有合法合理性**，而非平台单方面认定
3. **格式条款无效的法律后果是"该条款自始无效"**——平台不能主张第37条、第61-66条等免责条款来免除自身责任
4. **如果用户已付费购买课程**，平台封号后拒绝退款，用户可主张该用户协议中关于"违反协议不予退款"的条款因违反公平原则而无效

### 当你遇到账号封禁时的应对策略

> 注意：以下策略是基于前述法律分析提出的可能性，实际效果取决于具体情况和司法机关的判断。

| 场景 | 可能的应对方式 |
|------|--------------|
| 账号被以"违反协议"为由封禁 | 要求平台说明具体违反的是哪一条款，并提供该条款符合《民法典》第496条（合理提示说明义务）的证据 |
| 封号且拒绝退还已购课程费用 | 通过 **12315** 投诉，主张平台格式条款无效，要求按比例退还剩余未使用费用 |
| 平台主张第91条杭州管辖 | 消费者用户可在答辩期内提出管辖权异议，主张该格式管辖条款增加维权成本 |
| 平台依据第37条主张账号泄露不担责 | 若账号泄露系因平台安全漏洞导致，可主张第37条因违反《电子商务法》而无效 |
